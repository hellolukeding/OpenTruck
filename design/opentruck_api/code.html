Chat Completions

Given a list of messages comprising a conversation, the model will return a generated response. This endpoint provides access to OpenTruck's highest-throughput intelligence nodes.

POST https://api.opentruck.network/v1/chat/completions

Request Body

Parameter Description
model
string required
ID of the model to use. See the model endpoint compatibility table for details on which models work with the Chat API.
messages
array required
A list of messages comprising the conversation so far. Each message object requires a role (system, user, assistant) and content.
temperature
number optional
What sampling temperature to use, between 0 and 2. Higher values like 0.8 will make the output more random, while lower values like 0.2 will make it more focused and deterministic.

Defaults to 1.0.
stream
boolean optional
If set, partial message deltas will be sent, like in ChatGPT. Tokens will be sent as data-only server-sent events as they become available.
Example Request 
curl https://api.opentruck.network/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $OPENTRUCK_API_KEY" \
  -d '{
    "model": "ot-intel-v4",
    "messages": [
      {
        "role": "system",
        "content": "You are a helpful assistant."
      },
      {
        "role": "user",
        "content": "Initialize distribution sequence."
      }
    ]
  }'
